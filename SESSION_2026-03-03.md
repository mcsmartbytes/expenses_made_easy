## Code Changes (committed: `60b77ae`)

### 1. `capacitor.config.ts` — useLegacyBridge
- Added `useLegacyBridge: true` to the `android` section
- Required by `@capacitor-community/background-geolocation` to prevent location updates from silently stopping after ~5 minutes in the background

### 2. `utils/nativeMileageTracker.ts` — Crash Recovery
- Added `localStorage`-based trip persistence for crash/reload recovery
- `persistState()` — saves tracking state to `nativeMileageTracker_state` key on every location update and at trip start
- `restoreState()` — called during `initialize()`, restores an in-progress trip (24h staleness guard)
- `clearPersistedState()` — called in `stopTracking()` after trip data is captured
- On restore, fires `onAutoStart` callback so the UI re-enters tracking mode

### 3. `app/(expenses)/mileage/page.tsx` — Screen Wake Lock
- Added Wake Lock API for web-mode fallback (keeps screen on while tracking)
- `acquireWakeLock()` — called when web-mode tracking starts (not native)
- `releaseWakeLock()` — called on stop tracking and in cleanup effect
- Only activates in web mode; native tracking doesn't need it

## Database Fix — `po_number` Column (no commit, SQL only)

### Problem
Saving an expense after receipt OCR scan failed with: `could not find "po_number" column of "expenses" in the schema cache`

### Cause
The code in `app/expenses/new/page.tsx` references `po_number` on the `expenses` table, but the migration (`supabase_add_po_numbers.sql`) was never run against the database.

### Solution
Ran the following SQL in Supabase SQL Editor:
```sql
alter table if exists public.expenses add column if not exists po_number text;
create index if not exists idx_expenses_po on public.expenses(po_number);
```
